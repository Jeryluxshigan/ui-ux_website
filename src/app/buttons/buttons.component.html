Components / Buttons
Buttons
  • A button should be used in its default state.
  • If a button needs to be highlighted, then it should carry the “danger” class.
  • A 5px gap should be kept between buttons.
  • No button should be given a fixed size. All buttons should be responsive.
  • When the button action needs to be prevented, the button should be in an inactive state unless the error needs to be showcase to the user
  • Search / Save buttons should always be displayed at far right of the screen.
Example
<button kendoButton (click)="onButtonClick()" class="example-btn>
My Button
</button>
Button Group
  • Each Button in the ButtonGroup can be separately configured depending on the requirements of your project and according to its API reference.
Example
